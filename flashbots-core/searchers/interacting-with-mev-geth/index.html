interacting with mev-geth

MEV-Geth is a fork of the go-ethereum client that miners run to accept searchers' bundles. You do not need to run MEV-Geth as a searcher, but, instead, to monitor the Ethereum state and transaction pool for MEV opportunities and produce transaction bundles that extract that MEV. Anyone can become a searcher. In fact, the bundles produced by searchers don't need to extract MEV at all, but we expect the most valuable bundles will.

An MEV-Geth bundle is a standard message template composed of an array of valid ethereum transactions, a blockheight, and an optional timestamp range over which the bundle is valid.

You can access the mev-geth repository here.

MEV-Geth bundle selection logic#
